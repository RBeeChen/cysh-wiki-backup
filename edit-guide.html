① 註冊 GitHub 帳號

GitHub 是一個免費的程式碼與文件託管平台，旭陵維基備份的所有內容都放在上面。要參與編輯，您必須先擁有一個 GitHub 帳號。

1
前往 GitHub 註冊頁面
點擊下方連結，進入 GitHub 官方註冊頁面：

前往 GitHub 註冊頁面
2
填入基本資料
依序填入：
  • Email address：您的電子郵件
  • Password：至少 8 字元，包含數字與字母
  • Username：您在 GitHub 上的公開名稱（只能用英文、數字、連字號）
  • Email preferences：是否接受 GitHub 電子報，可以選 n
3
完成人機驗證
頁面會出現拼圖或圖片選擇的驗證，按照指示完成即可。
4
驗證電子郵件
GitHub 會寄送一封驗證信到您填入的 Email，點擊信件中的驗證連結，帳號即完成啟用。
💡 已有 GitHub 帳號的話，直接跳到下一節。

② Fork 本專案

「Fork」是將本專案複製一份到您自己的 GitHub 帳號底下，讓您可以自由修改， 而不影響原本的備份站。修改完成後再透過 Pull Request 提交給管理員審核。

1
前往本專案的 GitHub 頁面
點擊右上角的「GitHub」連結，或直接前往：

https://github.com/RBeeChen/cysh-wiki-backup
2
點擊右上角的 Fork 按鈕
在頁面右上角找到「Fork」按鈕（在 Star 旁邊），點擊它。
3
建立 Fork
頁面會跳到「Create a new fork」，保持預設值，直接點擊「Create fork」即可。 完成後您的帳號底下會出現一個 cysh-wiki-backup 的副本。
💡 Fork 只需要做一次。之後每次要新增或修改條目，都在您自己 Fork 的副本上操作即可。

③ 線上新增條目

GitHub 提供網頁版的編輯器，不需要安裝任何軟體，直接在瀏覽器中新增 .md 檔案。

1
進入您的 Fork 副本
前往 https://github.com/您的帳號/cysh-wiki-backup
2
進入 docs 資料夾
點擊專案中的 docs 資料夾，所有條目的 Markdown 檔案都放在這裡。
3
點擊「Add file」>「Create new file」
在 docs 資料夾頁面右上角，找到「Add file」下拉選單，選擇「Create new file」。
4
填入檔案名稱
在頂部的輸入框中填入檔名，例如：
熱音社.md　或　王小明老師.md

⚠️ 檔名即為條目標題，請盡量使用中文正式名稱，不要加入特殊符號。 老師條目請在名稱後面加上「老師」兩字，社團條目請加上社團類型（如「○○社」）。
5
撰寫內容
在下方的編輯區輸入條目內容，請使用 Markdown 格式（詳見第五節）。 建議開頭先用 # 條目名稱 作為標題。
6
送出變更
滾到頁面最下方，在「Commit changes」區塊填入簡短說明（例如：新增熱音社條目）， 然後點擊「Commit changes」。

④ 修改現有條目

1
在網站上找到條目
在本備份站使用右上角的搜尋功能找到要修改的條目，進入頁面後， 點擊頁面頂部的「編輯此頁」連結。
2
GitHub 會自動帶您到編輯介面
若您已登入 GitHub，頁面會跳到該條目對應的 .md 檔案， 點擊右上角的鉛筆圖示（✏️ Edit this file）。
3
修改內容並送出
修改完成後，滾到底部，填入修改說明，點擊「Commit changes」。

⑤ Markdown 撰寫基礎

Markdown 是一種簡單的標記語言，讓純文字也能有格式。以下是常用語法：

效果語法
大標題# 標題文字
中標題## 標題文字
小標題### 標題文字
粗體**粗體文字**
斜體*斜體文字*
超連結[顯示文字](https://網址)
無序清單- 項目一 - 項目二
有序清單1. 第一項 2. 第二項
引用> 引用文字
程式碼`程式碼`
分隔線---
💡 建議每篇條目最上方都用 # 條目名稱 開頭，並在下面簡短介紹這個條目是什麼。

⑥ 送出 Pull Request

完成新增或修改後，您的變更只在自己的 Fork 副本中， 必須透過「Pull Request」通知管理員，才能讓內容出現在正式備份站上。

1
前往您的 Fork 副本頁面
前往 https://github.com/您的帳號/cysh-wiki-backup
2
點擊「Contribute」>「Open pull request」
頁面頂部會出現黃色提示欄，說明您的 Fork 比原版多了幾個 commit， 點擊「Contribute」，再點「Open pull request」。
3
填寫 Pull Request 說明
填入標題（例如：新增熱音社條目）和說明，讓管理員知道您做了什麼修改。
4
送出
點擊「Create pull request」，管理員會收到通知並進行審核。 審核通過後，您的條目就會正式出現在旭陵維基備份！
💡 Pull Request 送出後，請耐心等待管理員審核。若有問題，管理員會在 PR 頁面留言與您溝通。

⑦ 常見問題

Q：我沒有 GitHub 帳號，可以直接提交內容嗎？

A：不行，GitHub 需要帳號才能進行任何修改。請依照第一節的步驟完成註冊，整個流程只需約 5 分鐘。

Q：我的 Fork 已經很久了，內容和原版不同步怎麼辦？

A：前往您的 Fork 頁面，點擊「Sync fork」>「Update branch」，即可將原版的最新變更同步到您的副本。

Q：我可以上傳圖片嗎？

A：可以，在 GitHub 的檔案編輯頁面，直接將圖片拖拉到編輯區即可上傳，系統會自動產生圖片連結。

Q：條目名稱有什麼規則嗎？

A：沒有硬性規定，但建議老師條目以「姓名老師」命名（如：王小明老師），社團條目以社團正式名稱命名（如：熱音社）。

Q：我送出 Pull Request 後多久會被審核？

A：視管理員的空閒時間而定，沒有固定時程。若超過一週未獲回應，可以在 PR 頁面底部留言詢問。
